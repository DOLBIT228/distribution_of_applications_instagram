# Distribution of Applications (Instagram)

Веб‑застосунок на **Streamlit** для автоматичного розподілу заявок з Bitrix24 між менеджерами за фіксованими правилами навантаження.

## 1. Що робить застосунок

Застосунок:
- авторизує користувача за логіном/паролем із `st.secrets`;
- підтягує заявки з Bitrix24 з конкретної воронки та статусу;
- автоматично розподіляє заявки між обраними менеджерами;
- переводить заявки у наступний/робочий статус;
- зберігає історію розподілу в локальній SQLite БД;
- показує щоденну таблицю розподілу;
- надсилає службові повідомлення (webhook або Telegram);
- підтримує паузу, коротку паузу для зміни менеджерів та повну зупинку.

---

## 2. Старт і запуск

### Вимоги
- Python 3.10+
- доступ до Bitrix24 webhook API
- налаштований `secrets.toml` у Streamlit

### Встановлення
```bash
pip install -r requirements.txt
```

### Запуск
```bash
streamlit run app.py
```

Після запуску відкрийте URL із терміналу (зазвичай `http://localhost:8501`).

---

## 3. Конфігурація (secrets)

Застосунок очікує такі секції в `st.secrets`:

- `auth.users` — список користувачів для входу:
  - `login`, `password`, `name`, `manager_id`
- `bitrix.webhook_url` — базовий URL вебхука Bitrix24
- `directions` — список напрямків:
  - `name`
  - `funnel_id`
  - `status_id` (звідки браємо заявки)
  - `in_progress_status_id` (куди переводимо в роботу)
  - `next_status_id` (fallback, якщо `in_progress_status_id` порожній)
  - `repeat_status_id` (опціонально: окремий статус для "ПОВТОРНЕ" / "БАЗА")
  - `batch_size` (ліміт активних заявок на менеджера)
  - `auto_interval_seconds` (інтервал автоциклу)
- `managers` — доступні менеджери:
  - `name`, `id`
- `chatbot` (опціонально):
  - `webhook_url` **або**
  - `telegram_bot_token` + `telegram_chat_id`
- `distribution.timezone` (опціонально, за замовчуванням `Europe/Kyiv`)

## 4. Логіка розподілу (детально)

### 4.1. Основний цикл
Коли натиснута кнопка **«Почати авто-розподіл»**, застосунок кожен цикл:
1. зчитує заявки зі статусу джерела;
2. рахує поточне навантаження в статусі "в роботі" для кожного активного менеджера;
3. визначає вільні слоти: `batch_size - поточні_активні`;
4. розподіляє заявки тільки між менеджерами з вільними слотами;
5. оновлює відповідального та стадію угоди в Bitrix24;
6. фіксує результат у `distribution_history`.

Якщо вільних слотів немає — цикл завершується попередженням без змін у Bitrix.

### 4.2. Типи заявок
В поточній логіці Instagram заявка класифікується так:
- `Консультація` — якщо `TITLE` починається з `КОНСУЛЬТАЦІЯ`;
- `Термін` — всі інші заявки з основного статусу;
- `Повторне` — заявки зі статусу `repeat_status_id`, де `TITLE` починається з `ПОВТОРНЕ`;
- `База` — заявки зі статусу `repeat_status_id`, де `TITLE` починається з `БАЗА`.

### 4.3. Порядок пріоритетів у циклі
У межах одного автоциклу порядок такий:
1. `Повторне` / `База` (зі `repeat_status_id`),
2. `Консультація`,
3. `Термін`.

Для `Термін` додатково враховується поле терміновості `UF_CRM_1749123119`:
- ближчим часом,
- завчасно,
- без терміну,
- на майбутнє.

### 4.4. Розподіл між менеджерами
- Для `Консультація` та `Термін` використовується **round-robin** (починаючи з менеджера після того, хто останнім отримав цей тип заявки).
- Для `База` (як fallback) використовується вибір менеджера з:
  1) найбільшим залишком вільних слотів,
  2) бажано не тим самим типом останньої заявки,
  3) мінімальною кількістю заявок цього типу за день.
- `Повторне`: якщо попередній відповідальний є у поточному активному списку і має вільний слот — заявка повертається йому.

### 4.5. Обмеження за номером телефону +1
Заявки з номером `+1` (перевірка в `TITLE`, `PHONE` і `UF_CRM_*`) **не беруться в розподіл до 17:00** за часовим поясом `distribution.timezone`.

---

## 5. Онбординг

Онбординг (модальне вікно з кроками та відео) **не відкривається автоматично при вході**.

Тепер він відкривається **лише вручну** через кнопку **«Як це працює»** у верхній частині екрану.

Медіафайли кроків мають лежати в каталозі `onboarding_media/` (`gif-1.webm` ... `gif-7.webm`).

---

## 6. Стани керування авто-режимом

- `stopped` — зупинено;
- `running` — активний авто-розподіл із циклічним перезапуском сторінки;
- `paused` — пауза (відновлення через кнопку запуску);
- `reconfiguring` — коротка пауза для зміни складу менеджерів.

При паузі/зупинці/зміні менеджерів користувач має вказати причину, після чого формується сервісне повідомлення.

---

## 7. Дані та зберігання

Локальна БД: `distribution_history.db`.

Таблиця:
- `distribution_history`:
  - `distribution_date`
  - `direction_name`
  - `manager_name`
  - `deal_type`
  - `deal_id`

На базі цієї таблиці будуються:
- щоденна зведена таблиця в UI;
- stop-report у бот при повній зупинці.

---

## 8. UX кнопок на екрані

- **Вийти** — очищає сесію користувача;
- **Як це працює** — відкриває онбординг;
- **Почати авто-розподіл** — запускає безперервний цикл;
- **Пауза** — ставить цикл на паузу (з причиною);
- **Пауза для зміни менеджерів** — змінює активний список менеджерів без повної зупинки;
- **Зупинити авто-розподіл** — повна зупинка (з причиною + звіт);
- **Оновити статус** — ручний rerun;
- **Очистити значення** — очищає історію поточного дня по поточному напрямку.

---

## 9. Типовий сценарій роботи

1. Увійти під своїм логіном.
2. Обрати напрямок.
3. Обрати менеджерів зміни.
4. За потреби відкрити **«Як це працює»** (онбординг).
5. Натиснути **«Почати авто-розподіл»**.
6. Контролювати таблиці:
   - розподілу за сьогодні,
   - кількість активних у роботі.
7. За потреби поставити паузу/змінити менеджерів.
8. Після завершення зміни натиснути **«Зупинити авто-розподіл»**.

---

## 10. Відомі нюанси

- Якщо `in_progress_status_id` і `next_status_id` не задані — розподіл блокується.
- Якщо Bitrix API повертає помилку — вона піднімається як `RuntimeError`.
- Якщо інтеграція бота недоступна — це не блокує основний розподіл (помилки нотифікацій "проковтуються").

---

## 11. Структура репозиторію

- `app.py` — вся бізнес-логіка та UI Streamlit;
- `requirements.txt` — залежності;
- `README.md` — документація;
- `onboarding_media/` — медіа для онбордингу (необов’язково в репозиторії);
